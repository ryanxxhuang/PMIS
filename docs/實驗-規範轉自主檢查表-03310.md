## 實驗結論

1. **可行性高**：單一章節即可抽出 30+ 條可量化檢查項，每條都有數值標準與出處條文，足以直接生成可列印的自主檢查表草稿。
2. **需人工覆核的點**：檢查時機分類（澆置前/中/後）是推斷的；部分條文含契約選項（如坍度規範之要求欄為空白待填）；頻率類規定（每日/每週試驗）應對映到「檢查頻率」欄位。
3. **產品化路徑**：沿用 parse-contract 模式——上傳規範 PDF → 按章節切塊 → LLM 抽出 `{檢查項目, 量化標準, 允許誤差, 檢查時機, 頻率, 出處條文}` → 工程師審核 → 存為該工項的檢查表範本 → 現場填實測值 → 對應 `/quality` 查驗紀錄。
4. **與工項樹的連結**：章碼（03310）可對映 PCCES `refItemCode` 前綴（0331xxx），檢查表範本可自動掛到對應工項。
