## ⛔ 1. Auth Redirect URLs 白名單(5 分鐘,不做=重設密碼/驗證信全斷)

忘記密碼與註冊驗證信的連結要導回 app,Supabase 只放行白名單內的網址。
**注意尾斜線坑**:app 送的是 `https://ryanxxhuang.github.io/PMIS/`(帶斜線),
白名單填無斜線的精確網址會不匹配而回退 Site URL——**一律用 `/**` glob**(本機實測踩過)。

- [ ] Supabase Dashboard → Authentication → **URL Configuration**
- [ ] Site URL 填:`https://ryanxxhuang.github.io/PMIS/`
- [ ] Redirect URLs 加:`https://ryanxxhuang.github.io/PMIS/**`
- [ ] 存檔後,到正式站用你自己的 email 走一次「忘記密碼」全流程驗證

(之後換自訂網域,這兩處要同步改。)

## ⛔ 2. 自訂 SMTP(不做=陌生用戶收不到任何信)

Supabase 內建信件服務**只會寄給專案團隊成員的 email、每小時個位數封**,
官方明言不供 production——真用戶註冊的驗證信與重設信會寄不出去。
Resend 的 API key 已在 secrets 裡(send-reminders 在用),但 SMTP 寄件人
需要**已驗證的網域**——所以這步和第 3 步綁在一起。

- [ ] 先完成第 3 步(網域)
- [ ] Resend Dashboard → Domains → 加你的網域,照指示到 DNS 加 DKIM/SPF 記錄,等驗證通過
- [ ] Supabase Dashboard → Authentication → **SMTP Settings** → Enable custom SMTP:
  - Host `smtp.resend.com`、Port `465`、Username `resend`、Password = Resend API key
  - Sender email `noreply@你的網域`、Sender name `PMIS AI`
- [ ] 用非團隊成員的 email(家人/第二信箱)註冊一次,確認驗證信寄達
