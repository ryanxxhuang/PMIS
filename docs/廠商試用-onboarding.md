# 施工廠商試用 — 上手指南

給第一批試用的施工廠商。分兩部分：**A. 管理員（你）發之前的前置設定**、
**B. 廠商第一次操作的走法**。最後有**建議試用路徑**與**回饋要問的問題**。

> App：<https://ryanxxhuang.github.io/PMIS/>（真實帳號、資料存 Supabase）
> 只想看功能不註冊：<https://ryanxxhuang.github.io/PMIS/demo/>（demo 模式，純展示）

---

## A. 管理員前置設定（發之前做一次）

對應 [`supabase/SETUP.md` §7 的 pre-flight checklist](../supabase/SETUP.md)，重點三步：

1. **關掉信箱驗證**（最省事）— Supabase 後台 *Authentication → Providers → Email →
   關閉 Confirm email*。廠商註冊後即時登入，不用等信。
2. **把 App 網址加進白名單** — *Authentication → URL Configuration* 的 Site URL 與
   Redirect URLs 填 `https://ryanxxhuang.github.io/PMIS/`。
3. **決定誰建案**：
   - **建議**：讓廠商自己註冊 → 自己建案 → 自己上傳標單。他是建立者(admin)，權限最完整，
     資料也都是他的（最貼近真實使用）。
   - 或你先建好案再把廠商 email 加進來——但目前**同一案的成員彼此能改對方資料、也能刪案**
     （SCOPE 缺口 #2），所以**這批 pilot 一案只放一家廠商**，先別把外部監造/機關拉進同一案。

> 提醒信（每日到期彙整）此階段預設只會寄到你自己的信箱；要真的寄到廠商需先在 Resend
> 驗證網域。pilot 不做也沒關係，見 SETUP §7.2。

## B. 廠商第一次操作（把這段直接轉給廠商）

### 1. 註冊 / 登入
開 <https://ryanxxhuang.github.io/PMIS/> → 「註冊」→ 填姓名、公司、
組織別選 **施工廠商**、職稱（如「工地主任」「品管工程師」）、email、密碼 → 建立帳號即登入。

### 2. 建立你的工程專案
第一次登入會引導建立專案：填工程名稱、契約編號、主辦機關、監造單位、地點、開工/竣工日。

### 3. 匯入標單（BOQ）— 系統的脊椎
左側 **標單 BOQ** → 上傳你的 **PCCES XML**（招標文件裡的電子標單）。
系統在瀏覽器裡直接解析成工項樹（可到數千項）。**成本、進度、品質、工安全部掛在這棵樹上**，
所以這步先做。

### 4. 每天：施工日誌
**施工日誌** → 新增當日日誌。可以：
- 手動填天氣（上午/下午）、出工人數、機具、材料、施工項目數量；或
- 用 **AI 現場辨識**：拍工地告示牌 / 白板照片，AI 自動幫你把欄位填好，你再校對。
- 需要送審或列印時，`/site-log/print` 出**工程會公定格式**（民國日期、開工後第 X 日曆天、
  累計自動彙計、簽章欄）。

### 5. 估驗計價 → 請款
**估驗計價** → 依工項填本期完成數量，系統自動算金額、累計、剩餘。→ **請款收款** 記錄
請款/收款狀態。→ **進度 S 曲線 / 排程 / 成本毛利** 會自動跟著動。

### 6. 品質與工安
- **品質**：三級品管、查驗；不合格自動開缺失（可拍照 AI 填缺失說明）。
  自主檢查表可帶量化標準、實測值自動判定。取樣試驗會自動追 7/28 天齡期。
- **工安**：四類工安紀錄。

### 7. 其他
- **變更設計**：上傳變更後 PCCES XML，自動 diff 出追加減明細。
- **契約**：上傳契約 PDF，AI 抽時程義務與罰則；到期項目彙整到 **提醒中心**。
- **施工月報**：自動帶入當月完成工項數量表、施工紀要，AI 起草檢討/下月計畫。
- **匯出**：各清單可匯出 CSV；Dashboard 可一鍵打包整案 JSON（資料隨時拿得走）。

## 建議試用路徑（第一週）

1. 註冊 → 建一個**正在做的真實案** → 上傳該案的 PCCES 標單。
2. 連續 3～5 天用**施工日誌**（其中至少一次試 AI 現場辨識拍照填單）。
3. 跑一期**估驗計價**，看 S 曲線 / 成本有沒有對上你心裡的數字。
4. 開一筆**查驗**，故意判不合格 → 看缺失自動開立的流程。
5. 出一次**施工日誌公定格式列印**，跟你現在手寫/Excel 的版本比。

這條路徑會踩過「BOQ → 日誌 → 估驗 → 品質 → 出表」整條主線，最能看出值不值得換掉現在的做法。

## 想從廠商收到的回饋（給你自己參考）

- **標單匯入**：PCCES XML 進得來嗎？工項對得上你們平常的分項嗎？
- **日誌**：AI 現場辨識填得準不準？省了多少時間？公定格式列印能不能直接交？
- **估驗/進度**：算出來的數字跟你們現在的對帳一致嗎？
- **最想要但現在沒有的一件事**是什麼？
- **願不願意付費**？一個月多少覺得合理？（驗證 SCOPE 方向 C：單邊付費意願）

把回饋收回來後，用它決定 SCOPE 第 4 節的方向（深化廠商 Web / 監造協作 / iOS 現場 App）。
