## 2. 摘要、上線建議與 exit criteria

本輪問題嚴重度：**P0 × 0、P1 × 2、P2 × 4**。

## 6. 問題清單（依嚴重度）

### P1-01｜排程日期 UI 當下存在，F5 與 DB 皆歸空

- **分類**：資料持久／optimistic 假成功；B4 指定修復未生效。
- **步驟**：A → 進度管制 → 找到 `壹.五 營造綜合保險費` → 填 planned start／finish → Tab 離欄 → 等待 → F5。換另一組日期重做一次。
- **證據**：兩次填值在當下 DOM 可見；無 loading、成功或錯誤訊息；F5 後皆空。`item_schedules.id=be0a882d-1562-4846-97a3-7fff01682e31` 存在，但 `planned_start=null`、`planned_finish=null`。
- **四重確認**：① 輸入後立即值正確；② 等待後仍顯示；③ 無任何錯誤／loading；④ F5 消失且 DB null。第二組日期同結果。
- **根因假設**：native date input 的 `onChange`／blur 未進入 async writer，或 writer 建了 row 卻沒有把 dates 帶入 update；UI 直接保留 uncontrolled DOM 值，沒有以 server response 校正。
- **建議修法**：把起迄日期綁 controlled state；blur/change 後 `await upsert(...).select()`，只有 server 回列才顯示成功；錯誤時 rollback 並紅字。加入 browser test：真實 date event → wait response → F5 → 查 `item_schedules`。

### P1-02｜可在無請款日、無收款日的狀態登錄實收，狀態仍為「待請款」

- **分類**：金流資料完整性／跨欄位狀態機。
- **步驟**：C → 請款收款 → 第 5 期輸入未來請款日與收款日（兩者被擋）→ 實收輸入 `9,999,999` → 通過「超過應領」二次確認 → 等待 → F5 → 匯出 CSV。
- **證據**：DB `invoice_date=null`、`paid_date=null`、`paid_amount=9999999`；UI 累計已收 9,999,999、未收 -9,999,999；CSV 第 5 期請款日／收款日空白、實收 9999999、狀態「待請款」。
- **根因假設**：三欄各自獨立 `onBlur` patch；`payStatus` 只看日期，沒有跨欄位 constraint。超額二次確認只檢查 `paid_amount > net`，未驗 invoice/paid date。
- **建議修法**：把「請款→收款」做明確狀態機：未請款不得填收款日或實收；實收非 null 必須同時有合法收款日，收款日必須不早於請款日。用單一 RPC／transaction 原子寫入金額與日期，DB 加 constraint/trigger；既有矛盾資料需 migration 稽核。加入 future-date 被擋後再填 amount 的 integration test。

### P2-01｜非草稿估驗仍顯示刪除鈕，雖然 DB 會阻擋

- **分類**：UI 權限／狀態 guard 不一致。
- **步驟**：A → 估驗 → 選第 4 期（監造審核）→ 檢查動作列。
- **證據**：仍有「刪除估驗期」；raw DELETE 得 HTTP 400，F5 後資料存在。其他已受理／已判定證據 UI 均已隱藏刪除。
- **根因假設**：估驗頁只判角色或選中期，沒有同時判 `status === '草稿'`；server trigger 已正確使用最新狀態。

- **建議修法**：mobile breakpoint 將主要與危險操作的可點區至少 44×44（文字可維持小）；首欄 `position: sticky; left: 0` 並處理背景／z-index；加入橫捲提示與 390px screenshot/target-size test。

### P2-03｜AI 多模組問題只回一個模組，沒有責任角色與下一步

- **建議修法**：先拆 intent 清單，再平行取 acceptance/sample/defect/valuation/ownership，各區明示「有資料／無資料／無權限」；最後生成責任人與 next action。對問題要求的每個 noun 做 coverage assertion。

### P2-04｜估驗列印把零保留款顯示為 `-0`

- **分類**：列印數值格式。
- **步驟**：開第 5 期估驗 → 列印估驗單 → Chrome 列印預覽 → 另存 PDF → render。
